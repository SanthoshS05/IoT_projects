Smart car parking - Simulation

Overview

This project aims to create a smart car parking system that can monitor and display the availability of parking spaces in real-time. The system uses an ultrasonic sensor to detect the presence of a vehicle in a parking spot, an ESP32 microcontroller to process the data, and the Blynk app to display the information on a smartphone.

Components

Ultrasonic Sensor (HC-SR04): Measures the distance to an object (car) and detects whether a parking spot is occupied.

ESP32: A powerful microcontroller with built-in Wi-Fi and Bluetooth, used to process sensor data and communicate with the Blynk app.

Blynk App: A mobile application that allows you to create a custom dashboard to monitor and control IoT devices.

How It Works

Sensor Detection: The ultrasonic sensor is placed in each parking spot to measure the distance to the nearest object. If the distance is below a certain threshold, it indicates that the spot is occupied.

Data Processing: The ESP32 reads the distance data from the ultrasonic sensor. If a car is detected, the ESP32 updates the status of the parking spot.

Data Transmission: The ESP32 sends the parking spot status to the Blynk server using Wi-Fi.

Real-Time Monitoring: The Blynk app displays the status of each parking spot in real-time. Users can see which spots are available or occupied on their smartphones.

Steps to Build the Project

Hardware Setup

Connect the ultrasonic sensor to the ESP32.

Power the ESP32 using a suitable power source.

Software Setup

Install the Blynk library in the Arduino IDE.

Write the code to read data from the ultrasonic sensor and send it to the Blynk server.

Upload the code to the ESP32.

Blynk App Configuration

Create a new project in the Blynk app.

Add widgets (e.g., LEDs, labels) to display the status of parking spots.

Link the widgets to the virtual pins used in the ESP32 code.

Smart water fountain - Simulation

Overview

This project aims to create a smart fountain system that can monitor and display water quality and level in real-time. The system uses an ultrasonic sensor to measure water level, a TDS sensor to measure total dissolved solids, and a turbidity sensor to measure water clarity. The ESP32 microcontroller processes the data and sends it to the Blynk app for real-time monitoring and control.

Components

Ultrasonic Sensor (HC-SR04): Measures the water level in the fountain.

TDS Sensor: Measures the total dissolved solids in the water, indicating water quality.

Turbidity Sensor: Measures the clarity of the water.

ESP32: A powerful microcontroller with built-in Wi-Fi and Bluetooth, used to process sensor data and communicate with the Blynk app.

Blynk App: A mobile application that allows you to create a custom dashboard to monitor and control IoT devices.

How It Works

Sensor Detection: The ultrasonic sensor measures the water level, the TDS sensor measures the total dissolved solids, and the turbidity sensor measures the water clarity.

Data Processing: The ESP32 reads the data from all sensors and processes it to determine the water quality and level.

Data Transmission: The ESP32 sends the processed data to the Blynk server using Wi-Fi.

Real-Time Monitoring: The Blynk app displays the water quality and level in real-time. Users can monitor the status of the fountain on their smartphones.

Steps to Build the Project

Hardware Setup

Connect the ultrasonic sensor, TDS sensor, and turbidity sensor to the ESP32.

Power the ESP32 using a suitable power source.

Software Setup

Install the Blynk library in the Arduino IDE.

Write the code to read data from the sensors and send it to the Blynk server.

Upload the code to the ESP32.

Blynk App Configuration

Add widgets (e.g., gauges, labels) to display the water level and quality.
